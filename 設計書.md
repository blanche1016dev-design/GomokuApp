# 五目並べアプリ 設計書

## 1. 目的

C# で動作する五目並べアプリを実装する。  
ユーザーは `exe` ファイルを実行して起動でき、CPU 対戦を行う。  
難易度は 3 段階用意し、最も簡単な難易度では「ユーザーが負けそうな局面」で推奨手を表示する。

## 2. 想定する提供形態

- 実行形式: Windows 用デスクトップアプリ
- 起動方法: ビルド成果物の `exe` を直接実行
- 開発言語: C#
- 想定フレームワーク: .NET + WinForms

### WinForms を採用する理由

- 単体 `exe` 配布との相性がよい
- 盤面 UI を比較的短期間で構築できる
- 学習コストと実装コストが低い

## 3. 対象ユーザー

- ローカル環境で気軽に五目並べを遊びたいユーザー
- AI 強度を変えながら練習したいユーザー
- 初級者向けに、危険局面での推奨手を参考にしたいユーザー

## 4. 機能要件

### 4.1 必須機能

- CPU 対戦ができる
- 難易度を 3 段階から選択できる
- 盤面に石を配置できる
- 勝敗判定ができる
- 対局のリセットができる
- 最も簡単な難易度でのみ、危険局面時に推奨手を表示できる

### 4.2 画面機能

- タイトルまたはメイン画面
- 難易度選択
- 先手/後手の選択
- 対局画面
- 勝敗表示
- 新しい対局の開始

### 4.3 推奨手表示機能

- 対象難易度: Easy のみ
- 表示条件:
  - 相手 CPU の次手または数手以内で明確に不利になる恐れが高い
  - ユーザー手番である
- 表示内容:
  - 推奨マスのハイライト
  - 短いメッセージ
    - 例: 「この位置に置くと相手の連続を防ぎやすいです」

## 5. ゲーム仕様

### 5.1 盤面

- 初期案: 15 x 15
- 各マスに黒石・白石・空のいずれかを保持する

### 5.2 勝利条件

- 縦・横・斜めのいずれかで 5 つ連続したら勝ち

### 5.3 対戦形式

- 人間 1 人 vs CPU

### 5.4 手番

- ゲーム開始時に先手/後手を選択可能とする案
- 先手は黒、後手は白を基本とする

### 5.5 禁止手

- 初期実装では採用しない案
- つまり連珠ルールではなく、標準的な簡易五目並べとして扱う

## 6. 難易度設計

### 6.1 Easy

- 目的: 初心者向け
- 思考方針:
  - 即勝ち手があれば優先
  - 即負けを防ぐ手があれば優先
  - それ以外は簡易評価で手を選択
- 支援機能:
  - ユーザーが危険な局面にある場合、推奨手を提示
- 想定実装:
  - 候補手を近傍マスに限定
  - 1 手先中心の軽量評価

### 6.2 Normal

- 目的: 一般ユーザー向け
- 思考方針:
  - 攻守バランス型
  - 複数候補の比較精度を Easy より上げる
- 想定実装:
  - 近傍候補 + 局面評価
  - 2 手程度を意識したスコアリング

### 6.3 Hard

- 目的: やり応えのある CPU
- 思考方針:
  - 勝ち筋形成と相手の脅威管理をより強く行う
- 想定実装:
  - Minimax + Alpha-Beta pruning をベースにする
  - 探索候補を近傍に絞り、応答時間を抑える
  - 評価関数で連の形を重視する

## 7. AI 設計

### 7.1 共通方針

- 全難易度で盤面候補を全マス探索せず、既存石の周囲のみ候補にする
- 次の優先順で着手候補を評価する
  - 自分の即勝ち
  - 相手の即勝ち阻止
  - 高評価パターンの形成
  - 中央寄り・既存石近傍の手

### 7.2 評価関数の観点

- 五連
- 開いた四
- 閉じた四
- 開いた三
- 閉じた三
- 二連
- 相手の脅威の遮断

### 7.3 推奨手ロジック

Easy 時のみ、ユーザー手番の開始時に以下を判定する。

- CPU に次手の強い勝ち筋があるか
- ユーザーが防御しない場合に高確率で不利になるか
- 防御または反撃として有効なマスがあるか

条件を満たした場合、最良の防御手を 1 箇所表示する。

## 8. 画面設計

## 8.1 画面一覧

1. スタート画面
2. 対局画面
3. 結果表示ダイアログ

## 8.2 スタート画面

- タイトル表示
- 難易度選択
  - Easy
  - Normal
  - Hard
- 先手/後手選択
- 対局開始ボタン

## 8.3 対局画面

- 盤面グリッド表示
- 現在手番表示
- 選択中難易度表示
- 推奨手表示エリア
- リセットボタン
- タイトルへ戻るボタン

## 8.4 表示ルール

- 置けるマスをクリック可能にする
- 最後に置いた石を視覚的に強調する
- 推奨手がある場合、枠線や半透明色で強調する
- 勝敗確定後は盤面入力を無効化する

## 9. 非機能要件

- 1 手あたりの CPU 思考時間
  - Easy: ほぼ即時
  - Normal: 0.5 秒以内を目安
  - Hard: 1.5 秒以内を目安
- ローカル実行のみで完結
- ネット接続不要
- 例外発生時もアプリが異常終了しにくい構成にする

### 11.2 CellState

### 11.3 GameSession

## 12. 処理フロー

### 12.1 ゲーム開始

1. ユーザーが難易度と先手/後手を選択
2. 初期盤面を生成
3. 先手が CPU の場合、CPU が初手を実行
4. 対局画面へ反映

### 12.2 ユーザー着手

1. ユーザーが空きマスをクリック
2. 合法手判定
3. 石を配置
4. 勝敗判定
5. 未終了なら CPU 手番へ移行

### 12.3 CPU 着手

1. 難易度に応じて `AiEngine` が候補手を計算
2. 最適手を返す
3. 石を配置
4. 勝敗判定
5. 未終了ならユーザー手番へ移行
6. Easy の場合のみ必要に応じて推奨手を更新

## 13. 例外・境界条件

- 盤面外クリックは無視
- 埋まっているマスへの着手は不可
- 既に終了した対局では新規着手不可
- 候補手が見つからない場合は空きマスから代替選択
- 引き分け条件
  - 全マスが埋まり、勝者なし

## 16. 確認したい仕様

以下は実装前に確定したい。

1. UI は `WinForms` で進めてよいか。`WPF` 希望なら差し替える。
2. 盤面サイズは `15 x 15` でよいか。`9 x 9` や `19 x 19` 希望があれば調整する。
3. 先手/後手はユーザーが毎回選べる仕様でよいか。
4. 禁止手なしの通常五目並べでよいか。連珠ルール希望なら設計を変更する。
5. 推奨手は 1 箇所のみ表示でよいか。複数候補表示が必要か。
6. 難易度名は `Easy / Normal / Hard` の英語表記でよいか。日本語表記にも変更できる。
