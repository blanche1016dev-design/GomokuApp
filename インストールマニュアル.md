# 五目並べ インストールマニュアル

## 1. 概要

このアプリは Windows 向けのデスクトップアプリです。  
`GomokuApp.exe` を実行して起動します。

主な機能:

- CPU 対戦
- 難易度 3 段階
- Easy での推奨手表示

## 2. 動作環境

- OS: Windows 10 / 11
- 画面: 一般的なフル HD 環境を推奨

### 補足

現在の配布物は `win-x64` 向けです。  
通常の 64bit 版 Windows PC で利用できます。

## 3. 一般ユーザー向けインストール手順

GitHub から配布ファイルを取得して起動する方法です。

### 手順

1. GitHub の公開ページを開きます。
2. `Releases` がある場合は、最新版の配布ファイルを開きます。
3. 配布ファイルをダウンロードします。
4. ダウンロードした zip ファイルがある場合は、任意のフォルダに展開します。
5. 展開先フォルダ内の `GomokuApp.exe` をダブルクリックして起動します。

### 推奨

- 展開先は `デスクトップ` や `ドキュメント` など、書き込み権限のある場所を使ってください。
- zip のままではなく、必ず展開してから実行してください。

## 4. GitHub からソース一式を取得して使う方法

配布済みの `exe` がない場合や、自分でビルドしたい場合の手順です。

### 方法 1: GitHub 画面からダウンロード

1. GitHub のリポジトリ画面を開きます。
2. `Code` ボタンを押します。
3. `Download ZIP` を選びます。
4. zip を任意のフォルダに展開します。

### 方法 2: Git で取得

`git` が使える場合は、次のように取得できます。

```powershell
git clone <リポジトリURL>
```

## 5. ソースから起動する手順

ソースコードから実行する場合は .NET SDK が必要です。

### 必要なもの

- .NET 9 SDK

### 起動手順

1. リポジトリを取得します。
2. ターミナルでリポジトリのフォルダへ移動します。
3. 次のコマンドを実行します。

```powershell
dotnet run --project .\GomokuApp\GomokuApp.csproj
```

## 6. ソースから exe を作成する手順

自分の環境で `exe` を出力したい場合の手順です。

### 必要なもの

- .NET 9 SDK

### 手順

1. リポジトリを取得します。
2. ターミナルでリポジトリのフォルダへ移動します。
3. 次のコマンドを実行します。

```powershell
dotnet publish .\GomokuApp\GomokuApp.csproj -c Release -r win-x64 --self-contained false
```

### 出力先

次のフォルダに `GomokuApp.exe` が出力されます。

```text
.\GomokuApp\bin\Release\net9.0-windows\win-x64\publish\
```

## 7. 起動方法

### 配布済み exe を使う場合

- `GomokuApp.exe` をダブルクリックします。

### ターミナルから起動する場合

```powershell
.\GomokuApp\bin\Release\net9.0-windows\win-x64\publish\GomokuApp.exe
```

## 8. 初回起動時の操作

1. 難易度を選択します。
2. 先手 / 後手を選択します。
3. `新しい対局を開始` を押します。
4. 盤面をクリックして石を置きます。

## 9. アンインストール

インストーラー形式ではないため、削除はフォルダ単位で行えます。

### 手順

1. アプリを終了します。
2. 展開したフォルダごと削除します。

## 10. よくある問題

### Q1. `GomokuApp.exe` が見つかりません

配布ファイルが zip の場合、展開後のフォルダを確認してください。  
ソース配布のみの場合は、`dotnet publish` を実行して `publish` フォルダを作成してください。

### Q2. 起動しても動きません

次を確認してください。

- Windows 64bit 環境で実行しているか
- zip を展開してから起動しているか
- 配布ファイル一式をまとめて同じフォルダに置いているか

### Q3. ソースからビルドできません

次を確認してください。

- `.NET 9 SDK` が入っているか
- コマンドをリポジトリのルートフォルダで実行しているか

SDK 確認コマンド:

```powershell
dotnet --version
```
